# 网站建设落地指南

这份文档用于规划一个真正能被他人访问、能注册登录、能发布内容、能管理数据的网站。重点关注安全、前端、后端、数据库、服务器五个部分。

## 1. 安全

安全不是上线后再补的功能，而是从账号、权限、数据、部署开始就要设计。

- 账号密码：密码必须加密保存，不能明文写入数据库。Django 默认会对密码做安全哈希，不能绕过 `create_user` 这类安全方法。
- 登录状态：用户登录后使用安全会话或 Token。一个账号只能同时登录一个设备时，需要在登录成功后清理旧会话。
- 权限控制：普通用户只能管理自己的内容；删除帖子、隐藏内容、管理用户必须限制给超级管理员或工作人员账号。
- 表单校验：注册、登录、发布、上传图片都要校验输入内容，防止空标题、非法文件、超大文件、异常请求进入系统。
- 文件上传：头像、车型图、帖子图只能允许图片类型，并限制大小；上传文件不能直接当代码执行。
- 后台保护：后台地址、管理员密码、权限分组要严格管理，超级管理员账号不能多人共用。
- CSRF/CORS：浏览器请求要设置跨站保护。正式上线时，接口只允许自己的域名访问。
- 敏感配置：数据库密码、邮箱密码、密钥不能写死在代码里，应放在服务器环境变量中。
- 日志审计：管理员删除内容、用户登录失败、上传失败等关键操作要保留日志，方便追查问题。

## 2. 前端

前端负责用户看到和操作的页面，目标是清晰、稳定、真实可用。

- 页面路由：每个按钮都要跳转到真实页面，不能只是停留在首页或做假交互。
- 数据来源：帖子、用户、车型、评论、关注、私信都应从接口读取，不能长期依赖写死的演示数据。
- 登录状态：页面要能识别游客和已登录用户。未登录时提示登录，登录后显示头像、昵称、粉丝、关注、等级。
- 发布体验：发布帖子后要立刻显示在列表和详情页中；如果关联了车型，也要出现在对应车型车友圈。
- 图片展示：车型图、头像、帖子图要支持后台上传，并处理图片无法显示时的兜底图。
- 错误提示：请求失败、登录失败、没有权限、发布失败时，要给用户能看懂的中文提示。
- 移动端适配：手机上导航、列表、弹窗、表单不能挤压或重叠。
- 公众文案：页面上不要出现框架名、接口、数据库、后端等技术词，除非是管理员工具。

## 3. 后端

后端负责业务逻辑、权限、接口和数据处理。

- 用户系统：支持注册、登录、退出、上传头像、查看当前用户。
- 内容系统：支持帖子、文章、车型、车友会、聚会、配件内容的创建、读取、更新、删除。
- 管理后台：超级管理员可以删除帖子、修改车型图片、管理用户、隐藏违规内容。
- 真实关系：粉丝数、关注数、等级、私信数量必须由数据库实时计算，不使用虚假的固定数字。
- 车型车友圈：每个车型应有专属页面，帖子通过关联车型进入对应车友圈。
- 接口规范：接口返回统一 JSON，包括成功数据和错误信息。前端不应该猜测后端结果。
- 数据校验：后端必须再次校验标题、正文、图片、用户权限，不能只依赖前端校验。
- 迁移管理：修改数据表后要创建迁移并执行迁移，保证数据库结构和代码一致。

## 4. 数据库

数据库是网站的真实数据来源，设计时要避免后期混乱。

- 数据表规划：用户、用户资料、关注关系、私信、车型、车款、帖子、文章、活动、配件等要分表管理。
- 关联关系：帖子可以关联车型、车友会、店家；车型可以拥有多个车款；用户可以关注其他用户。
- 唯一约束：用户名、车型 slug、文章 slug 等需要唯一，避免访问路由冲突。
- 数据状态：内容应有草稿、已发布、已隐藏状态，删除前可先隐藏。
- 备份策略：上线后数据库必须定期备份，至少保留最近多天备份。
- 图片存储：数据库只保存图片路径，图片文件放在媒体目录或对象存储里。
- 数据检查：上线前要检查是否存在空车型、重复 slug、缺失图片、孤立帖子、未迁移表结构。

## 5. 服务器

服务器负责让别人能访问网站，也负责稳定运行。

- 域名：购买域名后解析到服务器公网 IP。
- Web 服务：前端静态文件由 Web 服务托管，后端接口由应用服务运行。
- 反向代理：正式环境通常用 Nginx 或同类服务转发请求，例如 `/api/` 给后端，其它页面给前端。
- HTTPS：正式网站必须配置 HTTPS，保护登录密码和用户数据。
- 进程守护：后端不能靠手动命令窗口长期运行，应使用系统服务、进程管理器或容器自动拉起。
- 静态与媒体文件：静态资源、上传图片要配置正确访问路径。
- 防火墙：只开放必要端口，例如 80、443、管理端口；数据库端口不直接暴露给公网。
- 监控：监控网站是否可访问、服务器磁盘是否满、错误日志是否异常增长。
- 发布流程：本地开发完成后，先构建前端，再迁移数据库，再重启服务，最后检查首页、登录、发布、后台。

## 6. 建站顺序建议

1. 确定网站栏目和数据模型。
2. 建立后端项目、应用、数据库和管理员账号。
3. 建立前端项目和真实路由。
4. 接入注册、登录、发布、删除、图片上传等基础功能。
5. 将所有页面改为读取数据库真实数据。
6. 配置后台管理内容和权限。
7. 执行数据库迁移并检查数据完整性。
8. 本地完整测试首页、车型、车友圈、发布、登录、后台。
9. 部署到服务器，绑定域名和 HTTPS。
10. 上线后持续备份、监控、修复问题。
