# Tuner Hub 上线部署说明

## 推荐平台

先用 Render 上线。Render 可以连接 GitHub，并自动创建网站服务和 PostgreSQL 数据库。

## 已准备好的文件

- `render.yaml`：Render 部署配置。
- `build.sh`：安装依赖、构建前端、迁移数据库、收集静态文件。
- `Procfile`：云平台启动命令。
- `.gitignore`：避免上传本地数据库、虚拟环境和依赖目录。

## 上线步骤

1. 把项目上传到 GitHub。
2. 登录 Render。
3. 选择 New Blueprint。
4. 连接这个 GitHub 仓库。
5. Render 会读取 `render.yaml`，自动创建网站服务和 PostgreSQL 数据库。
6. 部署完成后，打开 Render 给你的网址。
7. 如果 Render 生成的网址不是 `https://tuner-hub.onrender.com`，需要在 Render 环境变量里修改：
   - `TUNERHUB_ALLOWED_HOSTS`
   - `TUNERHUB_CSRF_TRUSTED_ORIGINS`

## 需要确认的环境变量

- `DATABASE_URL`
- `TUNERHUB_SECRET_KEY`
- `TUNERHUB_DEBUG=0`
- `TUNERHUB_SECURE_SSL_REDIRECT=1`
- `TUNERHUB_HSTS_SECONDS=31536000`

## 上线后检查

- 首页能打开。
- 注册和登录正常。
- 后台 `/admin/` 能打开。
- 发帖正常。
- 我的车库可以添加车辆。
- 项目车记录可以添加记录。
- 上传头像和车型图片正常。
- 封禁账号正常。
- 手机端页面正常。

## 重要提醒

Render 免费服务的文件系统不是长期存储。用户上传的头像、车型图片等文件，后续建议接入云存储，例如 Cloudinary、S3 或其它对象存储。
